    通常，C语言中，sizeof(a)/sizeof(a[0]) 可以获取数组的长度，原理是 sizeof(a) 代表整个数组的大小，sizeof(a[0]) 代表数组中第一个元素的大小，而数组中的每个元素大小都是相同的，所以最后的计算结果就是数组的长度。
    
    但在这个程序里，b是指针型，sizeof(b)算的是sizeof(int*)而不是sizeof(int[])，32位下，指针型占8个字节，int占4个字节。和sizeof（a)是有区别的，陈学进老师的C语言课本P196的图9.26也对理解有帮助。
    
    本质上，翁恺老师在其C语言课程9.1的第4个视频中，把数组传进函数传的是指针（还不如传指针），所以在函数内无法用sizeof算数字内元素的个数。
